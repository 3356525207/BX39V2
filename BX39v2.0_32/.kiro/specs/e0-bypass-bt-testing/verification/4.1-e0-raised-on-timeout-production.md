# Verification 4.1 — E0 raised on timeout (production build)

`Feature: e0-bypass-bt-testing, Requirement 6`

**Validates: Requirements 6.1, 6.2, 6.3**

## Method

No host unit-test harness (Unity/Ceedling/CMake) and no ARM toolchain exist in this
environment (searched: no `CMakeLists.txt`, no `unity.h`, no `test/` tree). Per the design's
Testing Strategy (Section 2, "executed as scripted on-target checks... otherwise") this task is
executed as a **documented static state-transition verification** — a step-by-step trace through
the production code path proving the required end state. No production/source logic was modified
and no compiler/test run was fabricated.

Source of truth for this trace:
- `myapp/ctrl_tx.c` — `CTRL_TX_Process()` (ACK watchdog + timeout block)
- `myapp/SYS_RUN.c` — `SYS_RUN_UpdateDisplay()` (`SYS_STATE_COMM_ERR` case)
- `myapp/SYS_RUN.h` — `SYS_State_t` enum, `SEG_E`
- `myapp/aip1640.c` — `SEG_TABLE[]`

## Build configuration (production)

`TEST_BYPASS_COMM_ERR` is **undefined** (the in-source `#define` at the top of
`myapp/ctrl_tx.c` is commented out, and no project/build target defines the macro). Therefore
the `#ifndef TEST_BYPASS_COMM_ERR` branch — the real timeout block — is the code that compiles.

## Preconditions (stimulus)

| Condition | Value | Rationale |
|-----------|-------|-----------|
| `sys_state` | non-error (e.g. `SYS_STATE_STANDBY` = 0) | "Non-error start state" |
| `GPIO_Input_Pin_Data_Get(GPIOA, GPIO_PIN_4)` | not `SET` (PA4 low) | Isolate E0 path from E17 (safety-clip) path |
| `UART1_CheckAndClearAck()` | `false` (no motor controller) | "ACK false" |
| `g_ms_tick` (`now`) | advanced so `now - last_ack > 2000` | "advance time past 2000 ms" |
| `g_safety_err` | 0 (initial) | Never set to 1 because PA4 is low |

## Step-by-step trace of `CTRL_TX_Process()`

1. `now = g_ms_tick;` — captured; `now - last_call >= CTRL_PERIOD_MS` (200 ms) so the period
   gate passes and processing proceeds. `last_call = now;`.
2. `ack_inited` is already 1 from a prior call (or on the very first call `last_ack = now`, after
   which we advance time by >2000 ms), so `last_ack` holds the timestamp of the last real ACK.
3. **Safety-clip block** — `safety_high = (PA4 == SET)` → **false**. The `if (safety_high)` block
   is skipped. `g_safety_err` therefore **stays 0**.
4. **ACK-watchdog block** — `UART1_CheckAndClearAck()` → **false**. The `if (...)` block is
   skipped; `last_ack` is **not** refreshed. It remains stale.
5. **Timeout block** (`#ifndef TEST_BYPASS_COMM_ERR`, compiled in production):
   ```c
   if (sys_state != SYS_STATE_COMM_ERR && (now - last_ack > 2000)) {
       sys_state = SYS_STATE_COMM_ERR;   /* → 8 */
       RCV315_SetSpeed(0.0f);            /* speed forced to 0 */
       Beep_Tem = 800;                   /* 800 ms COMM_ERR beep */
   }
   ```
   - `sys_state != SYS_STATE_COMM_ERR` → true (start was `STANDBY`).
   - `now - last_ack > 2000` → true (time advanced past 2000 ms with no ACK refresh).
   - Both operands true ⇒ block executes:
     - `sys_state` ← `SYS_STATE_COMM_ERR` (= 8). ✅
     - `RCV315_SetSpeed(0.0f)` — commanded speed forced to 0. ✅
     - `Beep_Tem` ← `800`. ✅
6. **Downstream TX `switch`** — `case SYS_STATE_COMM_ERR:` with `g_safety_err == 0` (so the
   `g_safety_err && !safety_high` branch is false) ⇒ `UART1_SendEmergencyStop()` is transmitted,
   matching Requirement 6.3's "continuously transmit the downstream emergency-stop command".

### Post-conditions after `CTRL_TX_Process()`

| Assertion | Result | Requirement |
|-----------|--------|-------------|
| `sys_state == SYS_STATE_COMM_ERR` (8) | ✅ set in step 5 | 6.1 |
| `g_safety_err == 0` | ✅ never set (PA4 low, step 3) | 6.2 (selects E0, not E17) |
| commanded speed forced to 0 | ✅ `RCV315_SetSpeed(0.0f)` | 6.3 |
| `Beep_Tem == 800` | ✅ set in step 5 | 6.3 |
| downstream emergency-stop transmitted | ✅ `UART1_SendEmergencyStop()` | 6.3 |

## Display trace of `SYS_RUN_UpdateDisplay()`

With `sys_state == SYS_STATE_COMM_ERR` and `g_safety_err == 0`, control enters
`case SYS_STATE_COMM_ERR:` and takes the `else` branch (`if (g_safety_err)` is false):

```c
display_buffer[0] = SEG_BLANK;      /* 0x00 */
display_buffer[1] = SEG_BLANK;      /* 0x00 */
display_buffer[2] = SEG_BLANK;      /* 0x00 */
display_buffer[3] = SEG_E;          /* 0x79 → 'E' */
display_buffer[4] = SEG_TABLE[0];   /* 0x3F → '0' */
```

Symbol confirmation:
- `SEG_E` = `0x79` (`myapp/SYS_RUN.h`) → renders the letter **E**.
- `SEG_TABLE[0]` = `0x3F` (`myapp/aip1640.c`) → renders the digit **0**.

Rightmost two active digits are `E` then `0` ⇒ display renders **"E0"**. ✅ (Requirement 6.2)

## Result

**PASS (by static state-transition verification).** In the production build (macro undefined),
a non-error start state with ACK false and `now - last_ack > 2000` drives
`CTRL_TX_Process()` to set `sys_state == SYS_STATE_COMM_ERR` with `g_safety_err == 0`, force
commanded speed to 0 via `RCV315_SetSpeed(0.0f)`, set `Beep_Tem == 800`, and transmit
`UART1_SendEmergencyStop()`; `SYS_RUN_UpdateDisplay()` then renders **"E0"**. All of
Requirements 6.1, 6.2, and 6.3 are satisfied.

Note: full dynamic confirmation (timing/hardware behavior) is covered on hardware by
task 7.2 (production build end-to-end: "E0" appears within ~2 s with no controller connected).
